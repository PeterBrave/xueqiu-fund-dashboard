# 雪球基金组合工作台

一个本地 Chrome 插件 + 本地 AI 投研后端：把雪球/蛋卷里散落的基金按自定义 tag 聚合成组合，围绕「目标比例 vs 实时比例」做动态再平衡，并由阿里千问驱动的投研 Agent 基于每日仪表盘数据给出买入/卖出/加仓/调仓建议。

## 安装使用

### 方式一：直接下载插件包（推荐）

1. 到 [Releases](../../releases) 页面下载最新的 `xueqiu-fund-dashboard-x.y.z.zip`
2. 解压到任意目录
3. 打开 Chrome 的扩展程序页面：`chrome://extensions/`
4. 打开右上角「开发者模式」
5. 点击「加载已解压的扩展程序」，选择刚解压出来的目录
6. 打开雪球/蛋卷基金持仓页（danjuanfunds.com），点击浏览器插件图标，或页面里的「桌面看板」按钮

### 方式二：从源码构建

```bash
git clone <本仓库地址>
cd xueqiu
npm install
npm run build
```

然后按上面第 3-6 步操作，「加载已解压的扩展程序」时选择仓库里的 `dist/` 目录（或仓库根目录）。

### 启动 AI 投研后端

```bash
# 首次：配置 API key（已有 server/.env 可跳过）
cp server/.env.example server/.env   # 填入 DASHSCOPE_API_KEY

npm run server                        # 默认 http://127.0.0.1:8787
```

后端启动后刷新看板页，右栏「投研助手」会显示「AI 在线」。后端不在线时，对话自动回退到本地规则应答，看板其他功能不受影响。

## 页面结构

单页双栏，左边看数据、右边跟 AI 协作：

### 左栏 · 数据看板

1. **指标条**：总资产（按盘中估值实时估算）、盘中预估收益（含覆盖率与估值时间）、今日确认收益、持有收益
2. **组合比例 · 目标 vs 实时**（核心面板）：
   - 每个自建 tag 一行：实时市值、盘中涨跌、当前占比、目标占比、今日偏离、回到目标需要的买卖金额
   - 进度条上的黑色刻度是目标位置，偏离超过阈值的行会高亮
   - 「设置目标比例」打开专门的编辑器：滑杆 + 输入框设置各 tag 与现金的目标，实时显示已分配进度，总和必须为 100%（剩余部分自动归入「未分组」），支持「按当前比例填充」和「等比归一到 100%」
   - 每天自动留档一次各 tag 的实际比例与偏移量，「今日偏离」旁显示较昨日的变化，悬停可查看最近的偏移历史
3. **持仓明细**：按 tag 聚合、默认展开，每行带实时市值、实时占比、盘中估值涨跌、预估收益；点击基金名可跳转蛋卷基金详情页

### 右栏 · 投研助手

1. **AI 投研**（默认 tab）：一键生成今日投研报告——结论、市场观察、动作卡（买入/卖出/加仓/调仓/暂停定投，含金额与分批节奏）、定投节奏、风险提示；报告同时落盘为本地投研日志
2. **规则计划**：基于目标比例、再平衡阈值和单只上限自动产出动作卡 + 组合诊断（不依赖后端）
3. **AI 对话**：问组合偏移、调仓建议，也可以查基金概况、指数估值水位、券商研报；说「记住……」会写入本地长期记忆

## AI 投研后端（server/）

零依赖 Node 服务，`npm run server` 启动，配置在 `server/.env`（DashScope API key、模型、端口）。

### 投资策略（写死在系统提示词 + 可编辑的策略档案）

- **改良版全天候**：按 tag 目标比例做配置骨架，但再平衡永远顺着趋势做——欠配资产只在上行趋势（或企稳）时大额买入，下行趋势只用小额定投分批接；超配但强势的资产容忍超配，趋势转弱才兑现
- **Livermore 纪律**：突破/趋势确认后金字塔加仓，亏损头寸不加码，果断处理弱势品种
- **定投为主，趋势为辅**：趋势信号只决定定投的加减速

### 工作流程

`/api/advice`：接收仪表盘上下文 → 后端拉取每只持仓近 300 天净值，计算 MA20/60/120、20/60/120 日动量、距 52 周高点回撤和趋势判定（强势上行/上行/震荡/下行/弱势下行），再拉取蛋卷指数估值表（PE/PB 百分位）→ 连同策略档案与长期记忆一起交给千问 → 输出结构化 JSON 报告并写入投研日志。

### Agent skills（千问 function calling）

| skill | 说明 | 数据源 |
| --- | --- | --- |
| `fund_profile` | 基金概况：规模、经理、收益排名、股债仓位、前十大重仓 | 蛋卷 |
| `fund_nav_trend` | 净值走势 + 趋势指标 | 蛋卷 + 本地计算 |
| `fund_realtime_estimate` | 盘中实时估值 | 天天基金 |
| `index_valuation` | 指数估值表（PE/PB 百分位、低估/高估） | 蛋卷 |
| `broker_reports` | 券商研报：策略/宏观/行业/个股，含 PDF 链接 | 东方财富 |
| `search_fund` | 按关键词搜基金 | 天天基金 |
| `remember` | 写入长期记忆 | 本地 md |
| `recall_journal` | 回顾过往投研日志 | 本地 md |

### 本地记忆（`~/Documents/xueqiu-advisor-memory/`）

- `strategy.md`：策略档案，每次生成建议前完整读入，**可随时手动编辑**
- `notes.md`：长期记忆，AI 沉淀 + 手动补充
- `journal/YYYY-MM-DD.md`：每日投研日志，自动落盘

目录可在 `server/.env` 里用 `MEMORY_DIR` 更换。

### 接口

- `GET /api/health`：状态、模型、记忆目录
- `POST /api/advice`：生成今日投研报告（body: `{context}`）
- `GET /api/advice/today` / `GET /api/history`：查当日/历史报告
- `POST /api/chat`：对话（body: `{question, context, history}`）

## 前端接入点

`src/main.jsx` 中：

- `buildAgentContext()`：把 tag 比例、目标、偏离、盘中估值、持仓、偏移留档组装成结构化 JSON
- `askAgent(question, context, history)`：调 `/api/chat`，后端离线时回退本地规则
- `requestAiAdvice(context)`：调 `/api/advice` 生成整份投研报告

## 已实现

- 从 `danjuanfunds.com` 的持仓汇总接口读取数据，支持「全部基金账户」合并与子账户切换
- 盘中实时估值（`fundgz.1234567.com.cn`），比例、总资产、预估收益全部按估值实时计算
- 自定义 tag（分组）：手动指派、关键词自动分组、每个 tag 的目标比例全局持久化
- 规则调仓计划：偏离超阈值给出金额级买卖建议，单只基金超上限给出分散建议
- 交易记录接入：打开雪球交易记录页后自动缓存，进行中的交易会进入组合诊断
- AI 对话与 AI 计划的本地模拟 + Agent 接入点
- 搜索、粘贴 JSON 兜底导入，所有配置保存在浏览器本地

## 隐私说明

- 插件请求 `https://danjuanfunds.com/*`、`https://fundgz.1234567.com.cn/*` 和 `http://127.0.0.1/*`（本地投研后端）权限
- 持仓数据保存在 Chrome 本地存储；生成 AI 建议时，组合快照（比例、金额、持仓代码）会经由本地后端发送给阿里千问 API，不含账户凭证与 Cookie
- API key 只保存在 `server/.env` 本地文件，不进前端代码
- 只读取行情与持仓接口，不提供任何交易能力

## 计算口径

- 实时市值 = 确认市值 ×（1 + 盘中估值涨跌幅），无估值时用确认市值
- tag 当前占比 = tag 内基金实时市值合计 ÷（全部基金实时市值 + 现金宝）
- 未分组目标占比 = 100% − 各 tag 目标合计 − 现金目标
- 组合收益率 = sum(持有收益) / sum(市值 − 持有收益)
